# Release Standing — ggen v26.7.20 candidate

## Accepted source standing

- The 17-crate workspace builds and this repo's own `just pre-commit` gate chain is defined
  and runnable (fmt-check, check, lint, test-lib, coherence-check,
  guard-process-intelligence-boundary, guard-cheat-scan, guard-claims-schema, guard-pack-proofs).
- `just verify-tcps` (the self-hosted evidence loop over `examples/tcps-generated`) is a real,
  runnable command backed by a receipt chain (`ggen receipt verify`).
- `just docs-sync` re-syncs the root manifest and the Level Five Packs book and asserts the
  result is a content no-op (`git diff --quiet`); this pack's workflow makes that assertion a
  CI-enforced Andon rather than a manual habit.
- `scripts/ci/guard-publish-standing.sh` (this repo's no-overclaiming claims ledger gate) exists
  in both schema-only (pre-commit) and full/strict (pre-publish) modes and this pack's workflow
  runs both, in that order, before any tag is pushed.

## Standing NOT claimed

- **Binary standing** is `unverified-on-artifact-host`: this document is generated by `ggen sync`
  on whatever host runs the sync, and that host's build/test/clippy run is not, by itself,
  evidence a *tagged release* binary is sound -- that is what `.github/workflows/release.yml`
  (fired by this pack's tag-push step) establishes independently, per target, in CI.
- **L5 publication claim is WITHHELD.** ggen's own 第五水準 (Level Five) publication bar
  (GGEN-v26.7.20-製品要求書・構成設計書.md, 第二十九章, 23 conditions) requires, among other
  things, a current-generation-to-candidate-to-next-generation self-manufacture loop with a
  generation-2/generation-3 fixed point and an independent, dissimilar-toolchain double-check.
  None of that loop has been run and receipted for ggen itself as of this pack's generation.
  See `docs/PUBLICATION_JUDGMENT.md` for the full condition-by-condition table -- most
  conditions there are marked NOT MET. A green `publish-candidate` run and a green
  `release.yml` matrix build establish that the *candidate compiles, passes its own gates, and
  produces installable binaries* -- they do not, by themselves, establish the L5 self-manufacture
  claim.
