## 2. Logic Chain

1. **Premise 1**: For a round-trip parsing API, converting a struct `X` to string `S` via `to_string()` / `as_str()` and then parsing `S` via `from_string()` / `FromStr::from_str()` must yield the original struct `X` (or an equivalent one).
2. **Premise 2**: Let `X = AtomicPackId { class: AtomicPackClass::SurfaceMcp, name: "mcp" }`.
3. **Step 1 (Display)**: Under **Observation 1.1**, calling `X.as_str()` or `X.to_string()` formats as `"{class_str}-{name}"`. Since `class` is `SurfaceMcp`, its `class_str` matches as `"surface-mcp"`. Since `name` is `"mcp"`, the resulting string `S` is `"surface-mcp-mcp"`. This is corroborated by the test assertion in **Observation 1.3** which asserts `pack.to_string() == "surface-mcp-mcp"`.
4. **Step 2 (Parsing)**: Under **Observation 1.2**, parsing `S = "surface-mcp-mcp"` via `from_string` splits the string at the first hyphen. Thus `class_str = "surface"` and `name = "mcp-mcp"`.
5. **Step 3 (Matching)**: The `from_string` logic matches `class_str = "surface"`. Inside the nested match on `name`, it checks if the value matches `"mcp"` or `"a2a"`.
6. **Step 4 (Failure)**: Since `name` is `"mcp-mcp"` (and not `"mcp"` or `"a2a"`), the match returns `None`.
7. **Conclusion**: `AtomicPackId` does not round-trip. Any string generated by `.to_string()` or `.as_str()` is rejected by `.from_string()`, indicating a critical design error. This asymmetry also affects directory resolution where the registry looks for directories like `surface-mcp-mcp` instead of `surface-mcp`.

## 4. Conclusion

There is a critical API design and parsing asymmetry error in `crates/ggen-marketplace/src/marketplace/atomic.rs`. Specifically:
- `as_str()` formats `AtomicPackId` as `{class_str}-{name}` where `class_str` already contains the class suffix (e.g. `"surface-mcp"` + `"-"` + `"mcp"` = `"surface-mcp-mcp"`).
- `from_string()` expects a single hyphen to divide the namespace prefix (e.g. `"surface"`) from the name (e.g. `"mcp"`), and is incapable of parsing the resulting duplicate suffixes.
- This prevents `AtomicPackId` from round-tripping through string serialization and causes directory resolution issues in the pack resolver.
