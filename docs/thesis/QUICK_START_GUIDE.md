## Emergency Triage

**If time runs short, prioritize in this order:**

### Tier 1: Must Have (Defense Blockers)
1. Related Work chapter (without this, thesis will be rejected)
2. Research Methodology chapter (shows scientific rigor)
3. At least 2 additional case studies (validates generalizability)
4. Bibliography expansion to 60+ entries (minimum scholarly standard)
5. Limitations discussion (shows critical thinking)

### Tier 2: Should Have (Significantly Strengthens)
6. Formal definitions and theorems
7. Performance benchmarks
8. Comparison with competing approaches
9. Visual aids (diagrams, tables)
10. Future work expansion

### Tier 3: Nice to Have (Polish)
11. Executive summary
12. Index and glossary
13. Running example throughout
14. Navigation aids

**Cutting strategy:** If time is tight, do Tier 1 fully, Tier 2 partially, skip Tier 3.

## Common Pitfalls to Avoid

### Writing Pitfalls
- ‚ùå **Perfectionism:** Don't edit while drafting. Write first, edit later.
- ‚ùå **Scope creep:** Stick to the plan. Don't add unplanned sections.
- ‚ùå **Over-claiming:** Make testable claims backed by evidence.
- ‚ùå **Under-citing:** Every non-obvious claim needs a citation.

### Process Pitfalls
- ‚ùå **No backups:** Use Git. Commit every few hours.
- ‚ùå **No advisor check-ins:** Share outlines early, avoid surprises.
- ‚ùå **Skipping breaks:** Burnout reduces productivity. Take breaks.
- ‚ùå **Ignoring feedback:** If multiple reviewers say X, fix X.

### Technical Pitfalls
- ‚ùå **Untested examples:** Verify all code listings compile/run.
- ‚ùå **Broken references:** Check all \ref{} and \cite{} commands.
- ‚ùå **Inconsistent terminology:** Use search-and-replace for consistency.
- ‚ùå **Missing figures:** Reference all figures in text.
