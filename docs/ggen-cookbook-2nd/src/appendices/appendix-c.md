### Getting Help

#### Documentation Search
Before asking for help, check the documentation:
- Use the search function on [docs.ggen.dev](https://docs.ggen.dev)
- Browse the [cookbook](https://cookbook.ggen.dev) for examples
- Check the [CLI reference](./appendix-b.md)

#### Asking Good Questions
When asking for help, include:
1. **GGen version**: `ggen --version`
2. **Operating system**: Linux/Mac/Windows
3. **What you're trying to do**
4. **What you've tried**
5. **Error messages** (if any)

#### Reporting Bugs
When reporting bugs, include:
1. **Steps to reproduce**
2. **Expected behavior**
3. **Actual behavior**
4. **GGen version and environment**
5. **Minimal reproduction case**

## C.2 Learning Path

### Beginner Path

#### 1. **Getting Started** (1-2 hours)
- Read [Installation Guide](https://docs.ggen.dev/install)
- Complete the [Quick Start](https://docs.ggen.dev/quickstart)
- Try the [Interactive Tutorial](https://tutorial.ggen.dev)

#### 2. **Basic Templates** (2-4 hours)
- Learn [Template Basics](https://docs.ggen.dev/templates)
- Create your first template
- Understand [Frontmatter](https://docs.ggen.dev/frontmatter)

#### 3. **Marketplace Usage** (1-2 hours)
- Browse the [Marketplace](https://marketplace.ggen.dev)
- Install and use existing templates
- Understand [Template Variables](https://docs.ggen.dev/variables)

### Intermediate Path

#### 4. **Template Development** (4-8 hours)
- Study [Template Development Guide](https://docs.ggen.dev/gpack-development)
- Learn [RDF Integration](https://docs.ggen.dev/rdf-shacl-sparql)
- Create multi-file generators

#### 5. **Advanced Patterns** (8-16 hours)
- Read [Pattern Catalog](https://cookbook.ggen.dev/patterns)
- Study [Enterprise Patterns](https://cookbook.ggen.dev/part-6)
- Learn [Custom Processors](https://cookbook.ggen.dev/part-6/chapter-17)

#### 6. **Integration & Tooling** (4-8 hours)
- Set up [CI/CD Integration](https://docs.ggen.dev/ci-cd)
- Learn [IDE Integration](https://docs.ggen.dev/ide-plugins)
- Build custom integrations

### Advanced Path

#### 7. **Contributing** (16+ hours)
- Understand [GGen Architecture](https://docs.ggen.dev/architecture)
- Contribute to [Core Development](https://github.com/seanchatmangpt/ggen/blob/main/CONTRIBUTING.md)
- Write [Documentation](https://docs.ggen.dev/contributing)

#### 8. **Ecosystem Building** (20+ hours)
- Create [Template Packs](https://docs.ggen.dev/gpack-development)
- Build [Community Extensions](https://cookbook.ggen.dev/part-6/chapter-17)
- Establish [Best Practices](https://docs.ggen.dev/best-practices)

### Assessment Milestones

- **Beginner**: Can generate code from existing templates
- **Intermediate**: Can create and publish custom templates
- **Advanced**: Can contribute to GGen core or build complex integrations
- **Expert**: Can architect enterprise-scale code generation solutions

## C.7 Glossary

### Core Terms

**Code Generation**
: The process of automatically creating source code from higher-level specifications or models.

**Template**
: A reusable pattern that defines how to generate specific types of code with customizable parameters.

**Pattern Language**
: A structured vocabulary for describing solutions to recurring problems in a specific domain.

**Autonomic Computing**
: Self-managing systems that can configure, optimize, heal, and protect themselves.

**Frontmatter**
: YAML metadata at the beginning of a template file that defines variables, dependencies, and generation rules.

### Technical Terms

**RDF (Resource Description Framework)**
: A standard model for data interchange on the Web, used for representing information about resources.

**SPARQL**
: A query language for databases that store data as RDF triples.

**SHACL (Shapes Constraint Language)**
: A language for validating RDF graphs against a set of conditions.

**Marketplace**
: A curated ecosystem where developers can discover, share, and collaborate on code generation templates.

**Gpack**
: A package format for distributing collections of related templates with metadata and dependencies.

### Process Terms

**Deterministic Generation**
: The property that given the same inputs, a code generator will always produce identical outputs.

**Template Composition**
: The process of combining multiple templates to generate complex, multi-file outputs.

**Variable Resolution**
: The process of substituting template variables with actual values during generation.

**Validation**
: The process of checking generated code for correctness, style, and adherence to requirements.
