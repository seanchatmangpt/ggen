## Adding New Rules

### Rule Architecture

Each rule follows the `Rule<Q, T>` pattern:

```rust
pub trait Rule<Q: Queryable, T: Renderable> {
    fn execute(&self) -> Result<Vec<GeneratedFile>>;
}
```

### Step 1: Define Queryable

Create query executor in `src/codegen/rules/{rule_n}_*.rs`:

```rust
use ggen_codegen::Queryable;
use std::collections::HashMap;

pub struct Rule11Query;

impl Queryable for Rule11Query {
    fn execute(&self) -> Result<Vec<HashMap<String, String>>> {
        // Execute SPARQL query
        let results = vec![
            // Query results...
        ];
        Ok(results)
    }
}
```

### Step 2: Define Renderable

Create template renderer:

```rust
use ggen_codegen::Renderable;
use std::collections::HashMap;
use tera::Tera;

pub struct Rule11Template {
    tera: Tera,
}

impl Rule11Template {
    pub fn new() -> Self {
        let mut tera = Tera::new("templates/rule11/**/*.tera")
            .expect("Failed to load templates");
        Self { tera }
    }
}

impl Renderable for Rule11Template {
    fn render(&self, bindings: &HashMap<String, String>) -> Result<String> {
        let mut context = tera::Context::new();
        for (key, value) in bindings {
            context.insert(key, value);
        }
        self.tera.render("rule11.tera", &context)
            .map_err(|e| Error::Template(e.to_string()))
    }
}
```

### Step 3: Create Rule

Compose into `Rule<Q, T>`:

```rust
use ggen_codegen::Rule;

pub struct Rule11 {
    query: Rule11Query,
    template: Rule11Template,
}

impl Rule11 {
    pub fn new() -> Self {
        Self {
            query: Rule11Query,
            template: Rule11Template::new(),
        }
    }
}

impl Rule<Rule11Query, Rule11Template> for Rule11 {
    fn execute(&self) -> Result<Vec<GeneratedFile>> {
        let bindings = self.query.execute()?;

        let mut files = Vec::new();
        for binding in bindings {
            let content = self.template.render(&binding)?;
            let path = format!("rule11/{}.txt", binding.get("name")?);

            files.push(GeneratedFile {
                path: PathBuf::from(path),
                content,
            });
        }

        Ok(files)
    }
}

pub fn create_rule11() -> Result<impl Rule<Rule11Query, Rule11Template>> {
    Ok(Rule11::new())
}
```

### Step 4: Create Template

Create `templates/rule11/default.tera`:

```tera
// Generated by Rule 11
{{ name }}
{% for item in items %}
  - {{ item }}
{% endfor %}
```

### Step 5: Register Rule

Update `src/codegen/rules/mod.rs`:

```rust
pub mod rule11;
pub use rule11::create_rule11;
```

### Step 6: Create SPARQL Query

Create query file `queries/11-rule-description.rq`:

```sparql
PREFIX owl: <http://www.w3.org/2002/07/owl#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?item ?name
WHERE {
    ?item a owl:Class ;
           rdfs:label ?name .
}
```

### Step 7: Write Tests

Create test in `tests/rule11_test.rs`:

```rust
#[test]
fn test_rule11_creates_files() {
    let rule = create_rule11().expect("Rule 11 should create");
    let files = rule.execute().expect("Rule 11 should execute");

    assert!(!files.is_empty(), "Rule 11 should generate files");
    assert!(files[0].content.contains("Generated by Rule 11"));
}

#[test]
fn test_rule11_deterministic() {
    let rule = create_rule11().expect("Rule 11 should create");

    let files1 = rule.execute().expect("First execution");
    let files2 = rule.execute().expect("Second execution");

    assert_eq!(files1.len(), files2.len(), "Same file count");
    for (f1, f2) in files1.iter().zip(files2.iter()) {
        assert_eq!(f1.content, f2.content, "Identical content");
    }
}
```

## Code Style

### Rust Conventions

Follow Rust naming conventions:

- **Types**: PascalCase (`struct MyQuery`, `trait Queryable`)
- **Functions**: snake_case (`fn execute_query()`)
- **Constants**: SCREAMING_SNAKE_CASE (`const MAX_SIZE: u32`)
- **Modules**: snake_case (mod `my_module`)

### Formatting

```bash
# Format code
cargo fmt -p ggen-yawl

# Check formatting
cargo fmt -p ggen-yawl -- --check
```

### Linting

```bash
# Run Clippy
cargo clippy -p ggen-yawl -- -D warnings

# Fix auto-fixable warnings
cargo clippy -p ggen-yawl --fix
```

### Documentation

Document all public items:

```rust
/// Brief description (one line)
///
/// Longer explanation if needed (multiple lines)
///
/// # Examples
///
/// ```rust
/// let x = function_name();
/// assert_eq!(x, expected);
/// ```
///
/// # Errors
///
/// Returns error if condition fails.
pub fn function_name() -> Result<T> {
    // ...
}
```

## Testing Requirements

### Unit Tests

All rules must have unit tests:

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_rule_creates_files() {
        // Arrange
        let rule = create_rule();

        // Act
        let files = rule.execute().expect("Should execute");

        // Assert
        assert!(!files.is_empty(), "Should generate files");
    }
}
```

### Integration Tests

Add integration test in `tests/`:

```rust
#[test]
fn test_rule_integration() {
    let rule = create_rule().expect("Rule creation");
    let files = rule.execute().expect("Rule execution");

    // Verify files are valid
    for file in files {
        assert!(!file.content.is_empty());
    }
}
```

### Property Tests

Use proptest for edge cases:

```rust
use proptest::proptest;

proptest! {
    #[test]
    fn test_rule_with_random_input(s in ".*") {
        let rule = create_rule();
        let _ = rule.execute();  // Should not panic
    }
}
```

### Test Coverage

Aim for >80% code coverage:

```bash
# Run with coverage (requires tarpaulin)
cargo tarpaulin -p ggen-yawl --out Html
```

### Determinism Tests

Verify reproducibility:

```rust
#[test]
fn test_rule_deterministic() {
    let rule = create_rule();

    let result1 = rule.execute().expect("First run");
    let result2 = rule.execute().expect("Second run");

    // Compare outputs
    assert_eq!(
        hash(&result1),
        hash(&result2),
        "Same input should produce identical output"
    );
}
```

## Pull Request Process

### Before Creating PR

1. **Run all tests**
   ```bash
   cargo test -p ggen-yawl
   ```

2. **Check formatting**
   ```bash
   cargo fmt -p ggen-yawl -- --check
   ```

3. **Run linter**
   ```bash
   cargo clippy -p ggen-yawl -- -D warnings
   ```

4. **Verify documentation**
   ```bash
   cargo doc -p ggen-yawl --no-deps --open
   ```

5. **Run benchmarks** (if performance-related)
   ```bash
   cargo bench -p ggen-yawl
   ```

### PR Template

```markdown
## Description
Brief description of changes

## Type
- [ ] Bug fix
- [ ] New feature
- [ ] Enhancement
- [ ] Documentation
- [ ] Performance

## Related Issues
Closes #123

## Testing
- [ ] Unit tests added
- [ ] Integration tests added
- [ ] All tests passing

## Checklist
- [ ] Code formatted with `cargo fmt`
- [ ] Linter passes `cargo clippy`
- [ ] Documentation updated
- [ ] Tests verify changes
- [ ] No breaking changes
```

### PR Review Checklist

Reviewers check:
- ✓ Code quality and style
- ✓ Test coverage (>80%)
- ✓ Documentation completeness
- ✓ Performance SLOs met
- ✓ No breaking changes
- ✓ Determinism verified

## Common Issues

### Issue: "Template not found"

**Cause**: Template file path incorrect

**Solution**:
```rust
// Ensure template path matches actual file
let tera = Tera::new("templates/rule11/**/*.tera")?;
```

### Issue: "Query returns no results"

**Cause**: SPARQL query doesn't match ontology structure

**Solution**:
1. Test query in SPARQL editor
2. Verify ontology namespace prefixes
3. Check entity/property definitions

### Issue: "Tests fail intermittently"

**Cause**: Timing or ordering issues

**Solution**:
- Add proper synchronization
- Use deterministic test data
- Avoid external dependencies in tests
