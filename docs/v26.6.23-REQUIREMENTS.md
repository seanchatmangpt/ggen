## 1. WORKSPACE HEALTH (P0/P1 Blockers)

### P0 Critical Blockers (4 items)

| Blocker | Impact | Fix Effort | Status |
|---------|--------|------------|--------|
| **SHACL Validation No-Op** | ggen sync runs without semantic validation | 6h | SHACL gate always returns `pass` regardless of RDF violations |
| **Namespace URI Conflicts** | Duplicate URIs across ontologies; SPARQL fails | 4h | ggen-core allows conflicting `sh:class` definitions in pack-sourced queries |
| **CLI Error Type Chaos** | Exit codes inconsistent (2, 127, 1); unpredictable behavior | 5h | ggen-cli and ggen-core both use `i32` enums; no unified `ExitCodeMap` |
| **Package Conflict Detection Missing** | Two packs define same class; no early detection | 3h | PackRegistry::resolve() doesn't validate for namespace collisions |

**Total Effort**: 18 hours | **Path**: Fix in sequence (SHACL → namespace → error codes → conflict detection)

### P1 High-Priority Stubs (10 items)

10 critical crates have stub implementations that must either be completed or ship with explicit `not_implemented` error before release:

| Crate | Stub | Workaround | Impact |
|-------|------|-----------|--------|
| `ggen-lsp-a2a` | A2A agent bridge | Direct ggen-lsp-mcp calls | LSP agents cannot cross MCP boundary |
| `ggen-marketplace` | Pack discovery from remote URLs | Manual local pack dirs only | Cannot use published pack registries |
| `genesis-construct8` | Construct8 interop | No Construct8 support | Code-to-diagram synthesis impossible |
| `genesis-lockchain` | Distributed proof state | Centralized .ggen/receipts only | Cannot replicate proofs across nodes |
| `ggen-projection` | State projection to external systems | No external sync | Cannot push state to databases/APIs |
| `cpmp` | Symbol extraction for external tools | Manual SPARQL queries | IDE integration incomplete |
| `stpnt` | Stewardship cell governance | Hardcoded governance rules | Cannot customize cell behavior |
| `ggen-membrane` | Pack isolation layer | All packs share namespace | No security boundary between packs |
| `genesis-wasm-shell` | WASM/JS bindings | Node.js only (no browser) | Cannot use in browser environments |
| `ggen-a2a-mcp` | MCP server startup | Manual server launch required | Cannot auto-start MCP from CLI |

**Decision Required**: For each stub, decide:
1. **Complete it** (estimate 2-8h per crate)
2. **Ship with `not_implemented` error** (2h per crate, explicit in help text)
3. **Defer to v26.6.24** (document in ROADMAP.md)

**Default Recommendation**: Ship 5 high-value stubs with `not_implemented` error (ggen-marketplace, cpmp, ggen-projection, genesis-construct8, ggen-a2a-mcp); defer 5 others.

| Non-deterministic receipt IDs | UUID generated at render, not during load | 2h | ggen-receipt/src/lib.rs line 45 |
