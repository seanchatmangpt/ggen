
# 32. JSON SERIALIZER *


*The wire format is not the domain; the serializer is the translator.*


---

## Context

Jackson JSON serialization requires @JsonProperty, @JsonIgnore, custom serializers for types like Instant, UUID, and BigDecimal. These annotations pollute domain types. An @Entity class with @JsonProperty, @JsonIgnore, and @JsonSerialize annotations is serving two masters: the ORM and the HTTP layer. When the wire format changes, the entity class changes. When the entity changes, the serialization behavior changes.\n\nSeparating domain types from their wire representation using DTO records keeps both concerns clean. The DTO record is the wire format. The entity is the domain. The factory method (from) is the explicit translation between them.

---

❖ ❖ ❖

## The Problem

**Serialization annotations pollute domain types and create tight coupling to the wire format.**


The forces at play create tension between competing needs. Without resolution, these forces lead to suboptimal outcomes—complexity where simplicity is needed, fragility where robustness is required, opacity where clarity is essential.


---

## Therefore

Generate dedicated DTO records that handle all serialization concerns, keeping the domain types clean. Each java26:ValueObject or java26:AggregateRoot with java26:exposeAsRest true gets a generated DTO record with: @JsonProperty for field name customization, @JsonIgnore for internal fields, custom serializers for Instant (ISO-8601) and BigDecimal (string for precision), and a static from(Entity) factory method.\n\nThe DTO record is a snapshot of the entity at a point in time — immutable, serializable, and safe to pass across the HTTP boundary.

## Implementation in ggen sync


ggen generates a DTO record for every java26:AggregateRoot with java26:exposeAsRest true. Field type mappings: UUID → String (for JSON compatibility), Instant → String with ISO-8601 format, BigDecimal → String (to preserve precision in JSON), enums → their getValue() string. The from(Entity) factory is generated with the correct field mapping.
