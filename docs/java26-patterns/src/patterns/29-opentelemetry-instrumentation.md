
# 29. OPENTELEMETRY INSTRUMENTATION *


*Observability is not added later; it is generated with the code.*


---

## Context

Service-level observability requires span creation, metric recording, and log correlation. Adding these manually to every service method is tedious and inconsistent. Teams add spans to some methods and not others. Span names are inconsistent. Attributes are missing. The result is a tracing backend full of gaps and inconsistencies that make debugging harder, not easier.\n\nWhen observability code is generated alongside business code, it is consistent by construction. Every service method has a span. Every span has the same set of attributes. The trace is complete because the generator applied the same template to every method.

## The Problem

**Observability code added manually is inconsistent, incomplete, and often wrong.**


The forces at play create tension between competing needs. Without resolution, these forces lead to suboptimal outcomes—complexity where simplicity is needed, fragility where robustness is required, opacity where clarity is essential.


---

## Therefore

Generate OpenTelemetry instrumentation for every @Service method as part of the code generation. The ontology marks which operations are observable via java26:observable true. ggen generates @WithSpan annotations for simple cases and manual Span creation with error recording for complex cases. Every span records the operation name, aggregate class, and correlation ID from the ScopedValue request context (Pattern 33).\n\nMetrics are also generated: counters for operation invocations, histograms for operation duration. These are named consistently: {aggregate}.{operation}.count and {aggregate}.{operation}.duration.

## Implementation in ggen sync


ggen detects java26:observable true on java26:AggregateRoot and generates instrumented service wrapper classes. Simple methods receive @WithSpan. Methods that need custom attributes or error recording receive manual Tracer injection and Span lifecycle management in a try-finally block.
