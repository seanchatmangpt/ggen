
# 35. TEST SCAFFOLD *


*The test is the pattern's proof; generate both together.*


---

## Context

Generating code without generating tests leaves the generation unverified. Tests written long after the code are often superficial or wrong. They test what the code does rather than what the code should do. The test coverage reflects the code path, not the domain requirement.\n\nWhen tests are generated from the same ontology as the code, they verify the generated code against its specification. The test scaffold exercises every CRUD operation, validates the service layer contract, and confirms the REST controller returns the correct HTTP status codes. The test is the specification made executable.

## The Problem

**Tests written long after the code are often superficial or wrong.**


The forces at play create tension between competing needs. Without resolution, these forces lead to suboptimal outcomes—complexity where simplicity is needed, fragility where robustness is required, opacity where clarity is essential.


---

## Therefore

Generate JUnit 5 + AssertJ test scaffolds alongside every pattern implementation. The test scaffold validates the generated code's structure and behavior. Tests are generated from the same ontology as the code. Adding java26:testable true to any aggregate root triggers generation of: a unit test for the service layer (real dependencies, H2 in-memory DB), a MockMvc integration test for the REST controller, and a data builder class for test fixture construction.\n\nThe generated tests verify: create returns a non-null entity with the correct field values; findById returns empty for unknown IDs; update modifies only the declared fields; delete removes the entity; GET /api/{entities}/{id} returns 404 for unknown IDs.

## Implementation in ggen sync


ggen detects java26:testable true on java26:AggregateRoot and generates three test files: {ClassName}ServiceTest.java (unit test with @DataJpaTest), {ClassName}ControllerTest.java (MockMvc test with @WebMvcTest), and {ClassName}TestBuilder.java (test data builder using the builder pattern). All three are generated to src/test/java/{package}/.
