
# 11. PATTERN MATCHING VISITOR **


*The visitor no longer needs to be invited; Java 26 pattern matching brings it home.*

## Context

The Visitor pattern in Java 8 requires boilerplate: an interface with one visit() method for each type in the hierarchy, and an accept(Visitor) method in each concrete class. Adding a new operation means creating a new Visitor class. Adding a new type means updating every Visitor class. This double-dispatch ceremony obscures the actual logic in layers of indirection.\n\nJava 26 pattern matching for switch eliminates the ceremony. A switch expression on a sealed interface with type patterns is the Visitor pattern without the Visitor interface. Each case is one visit() method. The sealed hierarchy ensures all types are handled. The compiler enforces exhaustiveness.\n\nWhen ggen generates the sealed hierarchy, it can also generate the standard operations as switch expressions in a companion class. These generated operations serve as both documentation and a starting point for custom operations.

## The Problem

**Classical Visitor pattern requires modifications to the class hierarchy when adding new operations.**


The forces at play create tension between competing needs. Without resolution, these forces lead to suboptimal outcomes—complexity where simplicity is needed, fragility where robustness is required, opacity where clarity is essential.


---

## Therefore

Use Java 26 switch expressions with type patterns as a lightweight Visitor. The sealed hierarchy ensures exhaustiveness; the compiler detects missing cases.\n\nFor each java26:SealedInterface in the ontology, ggen generates a companion class named ${InterfaceName}Ops containing static methods for common operations (toString, toJson, validate, serialize). Each method is a switch expression over the sealed interface.\n\nUse when clauses in switch expressions for fine-grained dispatch: when an OrderStatus is a ShippedOrder with a specific carrier, the when clause can add carrier-specific logic without requiring a new sealed type.
