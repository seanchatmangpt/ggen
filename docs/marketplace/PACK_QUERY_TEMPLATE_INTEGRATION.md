# Pack Query and Template Integration (μ₂/μ₃)

## Overview

Packs can contribute SPARQL CONSTRUCT queries (μ₂) and Tera templates (μ₃) to the code generation pipeline. This enables packs to define custom extraction logic and code generation patterns.

## Architecture

```
μ₀ (Pack Resolution)
  ├─ Load pack queries from ~/.ggen/packs/<pack-id>/queries/*.rq
  ├─ Load pack templates from ~/.ggen/packs/<pack-id>/templates/*.tera
  └─ Return ResolvedPacks { queries, templates, merged_ontology }

μ₁ (Normalization)
  └─ Merge pack ontology with project ontology

μ₂ (Extraction)
  ├─ Extend with pack CONSTRUCT queries
  ├─ Execute all queries (standard + pack)
  └─ Generate IR graph (includes pack triples)

μ₃ (Emission)
  ├─ Extend with pack templates
  ├─ Render all templates (standard + pack)
  └─ Generate source files

μ₅ (Receipt)
  └─ Record pack provenance (queries_contributed, templates_contributed, digest)
```

## Pack Contract

See `docs/marketplace/PACK_QUERY_CONTRACT.md` for the full contract specification.

### Query Format (μ₂)

Queries MUST be CONSTRUCT-only (SELECT/ASK/DESCRIBE/UPDATE are forbidden).

```sparql
# ~/.ggen/packs/<pack-id>/queries/<name>.rq
PREFIX pack: <http://ggen.dev/pack/<pack-id>#>
PREFIX gen: <http://ggen.dev/gen#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

CONSTRUCT {
    ?entity gen:codeType "struct" ;
            gen:name ?name ;
            gen:packField ?field .
}
WHERE {
    ?entity a pack:TestEntity ;
            rdfs:label ?name .
    OPTIONAL {
        ?entity pack:testField ?field .
    }
}
```

### Template Format (μ₃)

Templates use Tera syntax with access to all query bindings.

```tera
# ~/.ggen/packs/<pack-id>/templates/<name>.rs.tera
// Auto-generated by {{ pack_id }}
// DO NOT EDIT

{% if name %}
pub struct {{ name }} {
    {% if field %}pub {{ field }}: String,{% endif %}
}

impl {{ name }} {
    pub fn new({% if field %}{{ field }}: String{% endif %}) -> Self {
        Self {
            {% if field %}{{ field }},{% endif %}
        }
    }
}
{% endif %}
```

### 2. ExtractionPass (μ₂)

```rust
// crates/ggen-core/src/v6/passes/extraction.rs

impl ExtractionPass {
    pub fn extend_with_pack_construct_queries(
        &mut self,
        queries: &[SparqlQuery],
    ) -> Result<()> {
        for sq in queries {
            // GATE: Verify CONSTRUCT-only
            if !sq.sparql.to_uppercase().contains("CONSTRUCT") {
                return Err(Error::new(&format!(
                    "Pack query '{}' is not CONSTRUCT; see docs/marketplace/PACK_QUERY_CONTRACT.md",
                    sq.name
                )));
            }

            // Add as TensorQuery with "pack::" prefix
            self.tensor_queries.push(TensorQuery {
                name: format!("pack::{}", sq.name),
                construct: sq.sparql.clone(),
                target_predicates: vec![format!(
                    "http://ggen.dev/v6/pack-query#{}",
                    sq.name.replace(['/', '\\', ' '], "_")
                )],
                order: base_order + 1 + i,
                description: Some(format!("Pack-contributed query {}", sq.name)),
            });
        }
        Ok(())
    }
}
```

### 3. EmissionPass (μ₃)

```rust
// crates/ggen-core/src/v6/passes/emission.rs

impl EmissionPass {
    pub fn extend_with_pack_templates(
        &mut self,
        templates: &[TemplateDef],
    ) -> Result<()> {
        for template in templates {
            // Extract rule name from template path
            let rule_name = format!("pack:{}::{}", pack_id, template_name);

            // Create binding key (pluralize)
            let binding_key = format!("{}{}", template_name, "s");

            // Add emission rule with inline template
            self.rules.push(EmissionRule {
                name: rule_name,
                template_path: PathBuf::from(&template.path),
                inline_template: Some(template.content.clone()), // Inline for reproducibility
                output_pattern: "{% if name %}{{ name | lower }}{% endif %}".to_string(),
                binding_key,
                iterate: true,
                skip_empty: true,
                description: Some(format!("Pack-contributed template from {}", path_str)),
            });
        }
        Ok(())
    }
}
```

### 4. Pipeline Orchestration

```rust
// crates/ggen-core/src/v6/pipeline.rs

impl StagedPipeline {
    pub fn run(&mut self) -> Result<BuildReceipt> {
        // μ₀: Pack resolution
        if let Some(ref resolver) = self.pack_resolver {
            let resolved = resolver.resolve()?;
            self.graph = resolved.merged_ontology.clone();
            self.resolved_packs = Some(resolved);
        }

        // Clone passes to avoid borrow issues
        let mut extraction = self.extraction.clone();
        let mut emission = self.emission.clone();

        if let Some(ref rp) = self.resolved_packs {
            // Load pack queries into μ₂
            extraction.extend_with_pack_construct_queries(&rp.queries)?;

            // Load pack templates into μ₃
            emission.extend_with_pack_templates(&rp.templates)?;
        }

        // μ₁: Normalization
        normalization.execute(&mut ctx)?;

        // μ₂: Extraction (includes pack queries)
        extraction.execute(&mut ctx)?;

        // μ₃: Emission (includes pack templates)
        emission.execute(&mut ctx)?;

        // μ₅: Receipt with pack provenance
        for pack_id in &resolved.atomic_packs {
            receipt.add_pack(PackProvenance {
                pack_id: pack_id.to_string(),
                version: resolved.pack_versions[&pack_id.to_string()],
                queries_contributed: resolved.query_names_for_pack(pack_id),
                templates_contributed: resolved.template_paths_for_pack(pack_id),
                digest: resolved.digest_for_pack(pack_id),
                // ...
            });
        }
    }
}
```

## Andon Gates

### μ₂: CONSTRUCT-Only Gate

Pack queries MUST be CONSTRUCT. SELECT/ASK/DESCRIBE/UPDATE are rejected:

```
🚨 SELECT Detected in μ₂:extraction

μ₂:extraction STOPPED THE LINE (Andon Protocol)

Query 'test-pack-integration::invalid-select' contains SELECT keyword without CONSTRUCT.

μ₂ is CONSTRUCT-only for IR generation. SELECT queries are forbidden.

Fix: Rewrite as CONSTRUCT query to generate IR triples, or move to μ₃.
```

### μ₃: Determinism Gate

Generated code must be deterministic (no timestamps, randomness, etc.):

```
🚨 Non-Deterministic Pattern Detected in μ₃:emission

μ₃:emission STOPPED THE LINE (Andon Protocol)

File 'output/entity.rs' contains non-deterministic timestamp pattern: 'Utc::now()'.

μ₃ output must be deterministic for reproducible builds.

Fix: Remove Utc::now() pattern or use deterministic alternative.
```
