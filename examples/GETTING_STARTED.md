# Getting Started with ggen v6.0.0

## Quick Overview

ggen is a **specification-driven code generation framework** that transforms RDF ontologies into reproducible code. This guide gets you up and running in 15 minutes.

**What You'll Learn:**
1. Project structure
2. Running examples
3. ggen workflow (spec → code → tests → deployment)
4. First code generation
5. Common first steps

## Prerequisites

**Required:**
- Rust 1.91.1+ (`rustup update`)

- Basic RDF/TTL knowledge (explained as needed)


## Understanding ggen Workflow

### Step 1: Write RDF Specification

**File:** `.specify/specs/MY-FEATURE/spec.ttl`

```turtle
@prefix : <http://example.com/my-feature/> .
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

:MyService a :RustService ;
    rdfs:label "My Service" ;
    :hasName "my-service" ;
    :hasPort 8080 ;
    :hasEndpoint :GetUser ;
    :hasEndpoint :CreateUser .

:GetUser a :HttpEndpoint ;
    rdfs:label "Get User Endpoint" ;
    :method "GET" ;
    :path "/users/{id}" ;
    :returns :User .

:User a :RustStruct ;
    rdfs:label "User Data" ;
    :hasField [
        :name "id" ;
        :type "uuid::Uuid" ;
        :required true ;
    ] ;
    :hasField [
        :name "name" ;
        :type "String" ;
        :required true ;
    ] .
```

**Why RDF?**
- Single source of truth
- Semantic relationships captured
- SPARQL queries for code generation
- Version control friendly
- Enables code archaeology

### Step 2: Create Tera Templates

**File:** `.ggen/templates/service.rs.tera`

```jinja2
// Generated from {{ service.name }} specification
// DO NOT EDIT - generated by ggen

use axum::{routing::get, Router};
use serde::{Deserialize, Serialize};

{% for field in service.fields %}
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct {{ field.struct_name }} {
    {% for attr in field.attributes %}
    pub {{ attr.name }}: {{ attr.type }},
    {% endfor %}
}

{% endfor %}

pub async fn create_router() -> Router {
    Router::new()
        {% for endpoint in service.endpoints %}
        .route("{{ endpoint.path }}", {{ endpoint.method_lower }}({{ endpoint.handler_name }}))
        {% endfor %}
}

{% for endpoint in service.endpoints %}
// Handler for {{ endpoint.label }}
pub async fn {{ endpoint.handler_name }}() -> impl axum::response::IntoResponse {
    // TODO: Implement handler
}

{% endfor %}
```

### Step 3: Run Code Generation

```bash
# Validate spec
ggen validate .specify/specs/MY-FEATURE/spec.ttl

# Generate code
ggen generate \
    --spec .specify/specs/MY-FEATURE/spec.ttl \
    --templates .ggen/templates/ \
    --output crates/generated/src/

# Verify generated code compiles
cd crates/generated && cargo check
```

### Step 4: Write Tests (Chicago TDD)

**File:** `crates/generated/tests/integration_test.rs`

```rust
#[tokio::test]
async fn test_get_user_endpoint() {
    // Arrange: Set up test data
    let app = create_router().await;
    let user_id = uuid::Uuid::new_v4();

    // Act: Call endpoint
    let request = Request::builder()
        .uri(format!("/users/{}", user_id))
        .method("GET")
        .body(empty_body())
        .unwrap();

    let response = app.handle(request).await;

    // Assert: Verify response
    assert_eq!(response.status(), StatusCode::OK);
    let body = response.into_body();
    assert_eq!(body.id, user_id);
}
```

### Step 5: Run Tests and Validate

```bash
# Run tests
cargo make test

# Check code quality
cargo make lint

# Verify SLOs (performance)
cargo make slo-check

# Full validation
cargo make pre-commit

# All must pass: Definition of Done
```

## Common Questions

**Q: Do I need to know RDF?**
A: It helps but not required. Start with examples, RDF will make sense quickly.

**Q: Can I use ggen without Rust?**
A: ggen generates code in any language (Rust, JS, YAML, etc.), but ggen itself is Rust.

**Q: How long until I'm productive?**
A: Most developers are productive in 2-3 hours of examples + documentation.

**Q: What if I break the build?**
A: `cargo clean && cargo make pre-commit` usually fixes it. Worst case: `git reset --hard origin/master`

**Q: How do I contribute?**
A: Fork repo, create feature branch, ensure `cargo make pre-commit` passes, submit PR.

## Summary

ggen workflow:
1. **Spec** (RDF) → Single source of truth
2. **Validate** (SHACL) → Catch errors early
3. **Generate** (Tera) → Deterministic code
4. **Test** (Chicago TDD) → Real collaborators
5. **Deploy** (ggen sync) → Cryptographic receipts

**Key principle:** Code is generated, not written. Specs are evolved, code follows.

Start with examples, consult ARCHITECTURE.md and PATTERNS.md for deep dives, and use TROUBLESHOOTING.md when things go wrong.
