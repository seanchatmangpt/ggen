# a2a-groq-agent

A2A agent powered by Groq LLM — generated by ggen from an RDF ontology.

This example demonstrates ggen's dog-fooding capability: ontology (.ttl) + SPARQL (.rq) + Tera template (.tera) = working A2A agent that calls Groq.

## Skills

| Skill | Description |
|-------|-------------|
| `generate-code` | Generate code from a natural language description |
| `explain-code` | Explain what a given piece of code does |
| `review-code` | Review code for bugs, style, and performance |
| `refactor-code` | Suggest refactored/improved version of code |

## Build

```bash
# Generate code from ontology
cd examples/a2a-groq-agent
ggen sync

# Build
cd generated
cargo build
```

## Run

```bash
# Set your Groq API key
export GROQ_API_KEY="your-key-here"

# Run the agent
cd generated
cargo run
# Listens on http://localhost:3000
```

## Architecture

```
ontology/a2a-groq-agent.ttl  →  SPARQL SELECT  →  Tera templates  →  Rust source
     (RDF/OWL)                   (ggen engine)     (.tera files)      (generated/)
```

The ontology defines the agent and its skills with system prompts. ggen executes SPARQL queries to extract skill metadata, then renders Tera templates that produce the final Rust code. The generated code uses a2a-rs for the A2A protocol and ggen-ai for Groq LLM calls.
