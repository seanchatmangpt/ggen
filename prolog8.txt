> **Prolog8 is a Rust/WASM proof engine for bounded rules, graph/bit execution, action admission, and replayable decisions.**


Modern systems increasingly take actions:


The organization later needs to know:

```text

```

Current answers are fragmented across:

```text

```

This creates an operational gap:

> **Decisions happen faster than organizations can prove why they happened.**

Prolog8 closes that gap.

---

# 5. Product Thesis

Prolog’s enduring value is:

> **Given facts and rules, derive what follows.**

But classic Prolog is too open-ended for hot-path infrastructure.

Prolog8 keeps the derivation power and removes the unstable execution surface:

```text
no parser in the kernel
no runtime code strings
no arbitrary scripting
no dynamic text evaluation
no unbounded rule bodies
no uncontrolled recursion
no proofless decisions
```

Instead:

```text
graph objects in
bits execute
proof out
```

# 6. Design Doctrine

## 6.1 Text Is Authoring, Not Execution

Text may exist in tooling.

Text may not execute inside the kernel.

```text
authoring layer:
  text allowed

admission layer:
  text becomes IDs, graph objects, fact blocks, rule objects, indexes

kernel:
  no strings
  no parser
  no runtime source evaluation

execution:
  graph/bit structures only
```

## 6.2 The Byte Is the Governor

Local hot-path surfaces must fit into byte-capped structures:

```text
arity ≤ 8
rule body atoms ≤ 8
variables ≤ 8
binding patterns ≤ 256
local operational states ≤ 256
feature mask = u8
negation mask = u8
built-in mask = u8
proof mask = u8
```

## 6.3 Need9 Means Split

If a construct needs more than eight primary elements, it is under-decomposed.

```text
Need9 arguments      → split predicate
Need9 body atoms     → introduce intermediate rule
Need9 variables      → split rule surface
Need257 states       → split rule family
Need dynamic text    → compile at boundary
Need mutation        → submit epoch delta
Need side effect     → boundary predicate with replay contract
```

## 6.4 Proof Is the Product

A boolean decision is incomplete.

Every serious decision should produce:

```text
answer
proof
negative proof if denied
facts used
rules fired
blocked alternatives
receipt
replay material
```

# 13. Functional Requirements

## FR-1: Kernel Must Reject Strings

The trusted kernel must not accept:

```text
query strings
predicate strings
term strings
runtime source text
textual meta-call
operator syntax
```

Kernel APIs must accept only IDs, byte buffers, graph objects, fact blocks, and query structs.

---

## FR-2: Kernel Must Execute `QueryAtom8`

A query atom must include:

```text
predicate ID
arity
binding mask
arguments
output mask
proof mode
```

The kernel must return:

```text
answer set
proof reference
receipt reference
execution status
```

---

## FR-3: Kernel Must Enforce Arity Cap

For Prolog8 hot tier:

```text
predicate arity ≤ 8
```

Any predicate exceeding the cap must be rejected before execution.

---

## FR-4: Kernel Must Enforce Rule Body Cap

For Prolog8 hot tier:

```text
rule body atom count ≤ 8
```

Exceeding rules must fail admission.

---

## FR-5: Kernel Must Enforce Variable Cap

For Prolog8 hot tier:

```text
variables per rule ≤ 8
```

---

## FR-6: Kernel Must Support Binding-Mask Dispatch

Each predicate must support access-path lookup by:

```text
predicate_id + binding_mask
```

Maximum binding patterns per predicate:

```text
2^8 = 256
```

---

## FR-7: Kernel Must Support Positive Proof

For `true` decisions, kernel must emit proof structure containing:

```text
facts used
rules fired
substitutions
proof node hashes
output root
```

---

## FR-8: Kernel Must Support Negative Proof

For `false` decisions, kernel must report at least one failure reason:

```text
missing fact
failed join
failed built-in
blocked by negation
expired epoch
revoked capability
source/root mismatch
```

---

## FR-9: Kernel Must Emit Receipts

Every decision must be able to produce:

```text
input root
rule root
fact root
proof root
output root
engine version
policy/catalog version
timestamp or epoch reference
```

---

## FR-10: Kernel Must Support Replay

Given a receipt and required input artifacts, the system must verify:

```text
same decision
same proof root
same output root
```

or return a clear mismatch.

---

## FR-11: Compiler Must Produce Admitted Graph/Bit Artifacts

Compiler output must contain no executable text authority.

It may include labels for display, but execution must reference IDs.

---

## FR-12: Doctor Must Explain Rejections

Doctor must return structured errors and repair hints.

Examples:

```text
E_ARITY_CAP_EXCEEDED
E_RULE_BODY_CAP_EXCEEDED
E_VARIABLE_CAP_EXCEEDED
E_STRING_QUERY_NOT_ADMITTED
E_UNSTRATIFIED_NEGATION
E_DYNAMIC_MUTATION_NOT_ADMITTED
E_FOREIGN_CONTRACT_MISSING
E_STATE_SURFACE_EXCEEDED
```

---

# 14. Non-Functional Requirements

Same admitted inputs must produce same outputs.

## NFR-2: No Hidden Runtime Parsing

Kernel must not contain a source parser path.

## NFR-3: Bounded Hot Path

Hot-tier constructs must fit declared caps.

## NFR-4: WASM Compatibility

Core execution must compile to WASM.

## NFR-5: Replayability

Receipts must be sufficient to verify decisions against referenced artifacts.

## NFR-6: Explainability

Proofs must be renderable into human-readable explanations outside the kernel.

## NFR-7: Small Embeddable Footprint

The product should be viable as:

```text
local library
edge module
browser-side WASM package
agent runtime plugin
workflow middleware
```

## NFR-8: No Internal JSON Representation

Boundary import/export may support JSON as external projection.

The core must not use JSON as internal state or execution representation.

# 1. Architecture Objective

Build a Rust/WASM proof engine that executes bounded rule graphs over admitted fact blocks using byte-capped operational surfaces.

The engine must derive decisions with proof and replay while refusing arbitrary code execution, text parsing, unbounded rule forms, and non-replayable side effects.

# 5. Feature Mask

Prolog8 hot tier defines at most eight admitted feature classes:

```text
bit 0 = facts
bit 1 = Horn rules
bit 2 = equality
bit 3 = typed comparisons
bit 4 = stratified negation
bit 5 = bounded recursion
bit 6 = controlled aggregates
bit 7 = contracted foreign predicates
```

If a construct has no admitted feature bit, it is rejected.

---

# 6. Admission Law

A program artifact is admitted only if:

```text
Admit(x) ⇔
  valid_shape(x)
  ∧ ids_exist_in_catalog(x)
  ∧ arity_within_cap(x)
  ∧ body_within_cap(x)
  ∧ variables_within_cap(x)
  ∧ feature_mask_admitted(x)
  ∧ negation_stratified(x)
  ∧ recursion_bounded_or_declared(x)
  ∧ predicates_indexable(x)
  ∧ builtins_typed(x)
  ∧ foreign_calls_contracted(x)
  ∧ proof_emittable(x)
  ∧ replay_deterministic(x)
```

# 8. Execution Model

## 8.1 Query Path

```text
QueryAtom8
  ↓

  ↓
lookup predicate
  ↓
derive binding mask
  ↓

  ↓

  ↓

  ↓
emit answer
  ↓

```

## 8.2 Binding-Mask Dispatch

For arity ≤ 8:

```text
binding_mask: u8
```

Each predicate may maintain:

```text
AccessPath[256]
```

Lookup:

```text

```

Access path types:

```text
ExactLookup
PrefixRangeScan
ColumnScan
IndexJoinSeed
FullScan

```

# 13. WASM ABI Requirements

The WASM interface should be byte-buffer oriented.

Example API shape:

```text

load_fact_block(ptr, len) -> FactBlockHandle
```

# 17. Decomposition Rules

Doctor must recommend decompositions.

## 17.1 Predicate Arity Split

Bad:

```text
event(case, activity, actor, resource, start, end, region, device, policy)
```

Good:

```text
event(case, event_id)
event_activity(event_id, activity)
event_actor(event_id, actor)
event_resource(event_id, resource)
event_time(event_id, start, end)
event_policy(event_id, policy)
```

## 17.2 Rule Body Split

Bad:

```text
allowed(X) :- a(X), b(X), c(X), d(X), e(X), f(X), g(X), h(X), i(X).
```

Good:

```text
allowed_precheck(X) :- a(X), b(X), c(X), d(X).
allowed_context(X)  :- e(X), f(X), g(X), h(X).
allowed(X)          :- allowed_precheck(X), allowed_context(X), i(X).
```

## 17.3 State Surface Split

If a rule family requires more than 256 local states:

```text
split into identity surface
capability surface
resource surface
epoch surface
evidence surface
```

# 24. Final Product Definition

Prolog8 is successful when a developer can embed it and say:

```text
Before this action happens, Prolog8 proves whether it is allowed.

If allowed, I get the proof.

If denied, I get the reason.

If audited later, I can replay the decision.
```

The core architecture remains:

```text
No parser in the kernel.
No strings in execution.
No unbounded rule surface.
No proofless decision.

Graph in.
Bits execute.
Proof out.
```
