## 2. Rust Enterprise Connector Pattern

All three connectors (SAP, Salesforce, ServiceNow) follow the same shape:

```rust
pub struct XxxConnector;

impl EventLogExtractor for XxxConnector {
    fn validate_config(config: &ConnectorConfig) -> Result<(), ConnectorError> { ... }
    fn extract(config: &ConnectorConfig) -> Result<ExtractionResult, ConnectorError> {
        // 1. Parse auth from config.params
        // 2. Build URL with pagination ($skip/$top, nextRecordsUrl, sysparm_*)
        // 3. HTTP GET/POST → deserialize JSON
        // 4. Map rows → EventLog via field_mappings
        // 5. Return ExtractionResult { log, metadata }
    }
}
```

**Key gotchas**:
- `log.traces.len()` — `traces` is a field, NOT a method (no parens)
- Closures that return `&str` fail lifetime inference → use `fn extract_str(v: &Value) -> Option<&str>` static method
- SAP `/Date(1234567890000)/` → `Utc.timestamp_millis_opt(ms).single()`
- ServiceNow timestamps: `NaiveDateTime::parse_from_str(ts, "%Y-%m-%d %H:%M:%S")`
- Trait methods (`EventLogExtractor::extract`) only callable with `use crate::connectors::EventLogExtractor;` in scope
- SAP/Salesforce Basic auth needs `base64 = "0.21"` in Cargo.toml; use `base64::engine::general_purpose::STANDARD.encode()`

```elixir

  Phoenix.PubSub.broadcast(OptimalSystemAgent.PubSub, "topic", msg)

cache_result =
```

### `@moduletag :requires_application` pattern
```elixir
# Any test that calls Registry.lookup, GenServer.call, or ETS tables
# owned by the app should be tagged so --no-start skips them cleanly
@moduletag :requires_application
```

### Task.async with bounded timeout (WvdA soundness)
```elixir
@fetch_timeout_ms 25_000

task = Task.async(fn -> fetch_process_metrics() end)
case Task.yield(task, @fetch_timeout_ms) || Task.shutdown(task) do
  {:ok, {:ok, metrics}} -> handle_success(metrics)
  {:ok, {:error, reason}} -> handle_error(reason)
  nil -> handle_timeout()
end
```

## 5. Cargo Test Hygiene: `#[ignore]` for Server-Dependent Tests

Tests that call `http://localhost:8090` (or any live service) should never run in CI without a server. Tag them:

```rust
#[test]   // or #[tokio::test]
#[ignore = "requires running pm4py-rust server on localhost:8090"]
async fn test_something_against_live_server() { ... }
```

This shows as `ignored` in `cargo test` output instead of `FAILED`, keeping the suite green.

Apply `#[ignore]` to all tests in files named `*_e2e_test.rs` or `*_integration_test.rs` that call live HTTP endpoints.

## 6. Cargo Clippy Common Fixes

| Warning | Before | After |
|---------|--------|-------|
| `manual_div_ceil` | `(n + k - 1) / k` | `n.div_ceil(k)` |
| `manual_clamp` | `x.max(lo).min(hi)` | `x.clamp(lo, hi)` |
| `needless_borrow` | `&String::from(...)` | `String::from(...)` |
| `unused_variable` | `let x = ...` | `let _x = ...` |

Always run `cargo clippy -- -D warnings` before committing. The `-D warnings` flag turns warnings into errors, matching CI behavior.

## 7. yawlv6 Hibernate Stale Session Pattern

After any rollback, clear the ThreadLocal session so the next `beginTransaction()` gets a fresh one:

```java
// In rollback handler:
} finally {
    closeSession();  // Unbind ThreadLocal — critical!
}

// In startTransaction(), guard against stale open sessions:
Session current = getSession();
if (current != null && current.isOpen()) {
    Transaction tx = current.getTransaction();
    if (tx != null && !tx.isActive()) {
        closeSession();  // Close stale session before opening new one
    }
}
```

**Pattern**: Distinguish transient (session threading) vs fatal (DB corruption) exceptions before setting `enginePersistenceFailure = true`.

## 9. Submodule Commit Ordering

When multiple submodules have changes + root needs pointer updates:

```
1. Commit in each submodule individually (cd into each, git add/commit)
2. cd back to root
3. git add <submodule1> <submodule2> ...
4. git commit -m "chore: sync submodule pointers ..."
```

Root commit will fail if any submodule has uncommitted working-tree changes (`m` lowercase in `git status --short`). Always clear each submodule fully before the root pointer commit.

**Diagnostic**: `m submodule` (lowercase) = uncommitted changes in submodule working tree. `M submodule` (uppercase) = staged pointer update. Both need resolution before root commit.
