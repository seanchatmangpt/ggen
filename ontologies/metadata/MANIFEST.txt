USAGE EXAMPLES
================================================================================

1. LOAD IN GGEN
   Edit ggen.toml:
   [ontology]
   imports = [
       "ontologies/metadata/dublin-core-terms.ttl",
       "ontologies/metadata/dublin-core-type-vocabulary.ttl"
   ]

2. SPARQL QUERY EXAMPLE
   PREFIX dcterms: <http://purl.org/dc/terms/>
   PREFIX dcmitype: <http://purl.org/dc/dcmitype/>
   
   SELECT ?resource ?title ?type
   WHERE {
       ?resource dcterms:title ?title ;
                 dcterms:type ?type .
   }

3. RDF TRIPLE EXAMPLE
   <http://example.org/doc/001>
       dcterms:title "Sample Document" ;
       dcterms:creator <http://example.org/person/alice> ;
       dcterms:type dcmitype:Text ;
       dcterms:issued "2024-01-15"^^xsd:date .

4. LOAD WITH OXIGRAPH
   oxigraph = Oxigraph()
   oxigraph.load("dublin-core-terms.ttl", format="turtle")
   oxigraph.load("dublin-core-type-vocabulary.ttl", format="turtle")

HISTORICAL CONTEXT
================================================================================

Timeline:
  1995-1997: Dublin Core Element Set development
  1998-07-02: Original 15 elements finalized
  2000-11-17: DCMI Type Vocabulary created
  2012-06-14: DCMI Metadata Terms (dcterms:) finalized
  2020-01-20: Latest namespace snapshot (current version in these files)

Purpose:
  The Dublin Core metadata vocabulary provides a standardized, cross-domain
  set of metadata terms for describing resources. Originally designed to
  enable discovery of web resources, it has grown into a comprehensive
  ontology used across digital libraries, archives, repositories, and
  semantic web applications.

Status:
  - Maintained by the Dublin Core Metadata Initiative
  - Widely used in library, museum, and archival communities
  - Foundation for many specialized metadata schemes
  - Integral to linked data and semantic web standards
  - Active community maintaining and extending the vocabulary
