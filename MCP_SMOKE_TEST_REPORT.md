## Root Cause Analysis (5 Whys)

### Problem
Generated `main.rs` contains `#[tokio::main]` (outer attribute) but test expects `#![tokio::main]` (inner attribute).

### 5 Whys Analysis

1. **Why did the test fail?**
   - Test assertion at line 234-236 expects `#![tokio::main]` but generated code has `#[tokio::main]`.

2. **Why is there a mismatch?**
   - The `render_main_template()` function (line 374-399 in `crates/ggen-core/src/mcp/mod.rs`) generates `#[tokio::main]` (outer attribute).

3. **Why does the template use outer attributes?**
   - Template author used the more common outer attribute syntax (`#[...]`), which is functionally equivalent for `tokio::main`.

4. **Why does the test expect inner attributes?**
   - Test author wrote assertion expecting inner attributes (`#![...]`), which are more idiomatic for file-level attributes.

5. **Why wasn't this caught earlier?**
   - This is a newly implemented MCP generation feature; integration test expectations don't match implementation.
