# Mermaid描画器出典調査

調査基準日は二〇二六年七月二〇日です。

## 結論

Mermaidには「全図種を一つの巨大な描画器が描く」という構成はありません。
中央の登録・検出・遅延読込路があり、各図種の`DiagramDefinition`が固有の
`parser`、`db`、`renderer`、`styles`を提供します。

中央路は次です。

1. `diagram-orchestration.ts`が組込検出器を列挙します。
2. `diagramAPI.ts`の`registerDiagram`が図種を登録します。
3. `Diagram.fromText`が型を検出し、必要ならloaderを実行します。
4. 読み込まれた定義から`renderer`を取得します。
5. `Diagram.render`が`renderer.draw(text, id, version, diagram)`を呼びます。

## 調査上の注意

`ontology.ttl`の`sourceConfidence`は、上流の検出器および図定義のimportを
直接確認した結果を記録します。共有描画器または方向別描画器を持つ図種は、
単一描画器へ丸めず、その関係を明示します。

同梱の`resolve-renderers.mjs`は、将来の上流commitでも登録路を再走査し、
検出器、定義、描画器の実在ファイルをHTTPで確認して確定JSONを生成します。
